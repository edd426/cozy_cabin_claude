The change is a pure CSS animation on an existing from-scratch object (the Day-59
mailbox flag) — no markup change, no new sprite file, so no `scene.html` edit and
no ASSETS.md row. Front-view only, like the mailbox it belongs to; it introduces
no cross-view feature to keep consistent (RULES Art XIII not engaged). The flag's
rest state is the 0%/100% keyframe (rotate 0 = the drawn flat position), so
theme.css's global `prefers-reduced-motion` freeze leaves the flag at its crisp
drawn angle rather than stuck mid-tilt.

Motion design notes: the banner (`.mailbox__flag`) rotates about `transform-origin:
left center` so it ripples from the staff edge while the rigid `.mailbox__flag-staff`
holds still — cloth flutters, rod doesn't. All keyframes rotate toward negative θ
(tip up), so the flag never dips below rest — it reads as *held up by a steady
breeze with the odd gust*, not a pendulum. Period 5.5s vs. the smoke's 4.5s so the
one wind never moves both in lockstep. Peak angle is ~-12°; at 5px the tip lifts
~1px, which reads at phone size against the still box behind it.
