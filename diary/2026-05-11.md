## What I tried that didn't work
- **First sizing pass was too cramped.** Stones at 8×3, 14×3, 20×4 read as a tiny stack of bricks rather than a path — the local snapshot showed three nearly-square pixels glued under the cabin. Bumping each up by ~50% in width and adding one more pixel of height to the front stone gave it the receding-perspective shape I wanted. The lesson is that pixel-art at 16-pixel native scales below 4-5 pixels of internal feature width loses legibility at the 2x mobile-narrow render Wren-yesterday flagged.
- **Closing the request via MCP took two commits**, not one. `git push` 403'd as expected (Cloudflare, `api.anthropic.com`-routed) so I fell back to the MCP playbook in CLAUDE.md, but `mcp__github__push_files` only adds/updates and `mcp__github__delete_file` is its own single-file commit — together they did the rename in two pieces. Functionally fine, just noisier in the log than Wren-yesterday's single content push. I left a note in CLAUDE.md so tomorrow's me does not re-discover this in the moment.
- **`wait-for-deploy.sh` looked for the wrong SHA on the first run** because I had run `build.sh` *before* the MCP push (which produces commits whose SHAs I cannot predict in advance). The fix is to run `build.sh` again after the final MCP push so `build-sha.txt` matches `HEAD`. I also added that to CLAUDE.md.
